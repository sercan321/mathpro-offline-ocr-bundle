# V172-Q335 Native Package Hash Evidence Collection / User-Side Native Package Intake Audit

Phase: `V172-Q335-NATIVE-PACKAGE-HASH-EVIDENCE-COLLECTION-USER-SIDE-NATIVE-PACKAGE-INTAKE`
Source phase: `V172-Q334-ACTUAL-BINARY-BUNDLE-JNILIBS-PADDLEPREDICTOR-PENDING-EVIDENCE-BLOCKED-GATE`

## Scope

Q335 adds a default-off native package hash evidence collection/intake gate for future Paddle Lite Android binary packaging. It prepares the evidence path and user-side intake script only.

## Guardrails

- No `.so`, `.jar`, or `.aar` bundle.
- No `jniLibs` or `android/app/libs` creation.
- No Gradle, pubspec, or AndroidManifest mutation.
- No `System.loadLibrary`.
- No Paddle Lite instantiation.
- No model loader, runtime startup, OCR, image-to-LaTeX, MathLive review, or workspace import.

## Required future evidence

- User-side native package files outside project root.
- `q328_native_library_package_hash_evidence.json` generated by Q328 capture script.
- Explicit user approval before any binary files are copied into the project.
